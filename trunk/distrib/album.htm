"CD Album" version 1.5

Congratulations! You now own a copy of the "CD Album" freeware. :o) The main purpose of this software is to serve as a player for CD albums, which are recorded on Data CDs. A CD album (in our terms) is a collection of musical albums saved in MPEG-3 format. Usually a musical CD can hold only one album, but a data CD can hold up to 10-12 albums in MPEG-3 format!

Usually CD albums are CDs that automatically start to play when you insert them into your CD drive. CD albums created for use with our software can come with song lyrics, pictures and other information, nicely designed in HTML. HTML has become a standard for designing documentation, and/so you will have a lot of freedom to design your musical CD albums.

You can also use "CD Album" to browse and play MPEG-3 files, which are stored on your hard disks or CDs. As a player, "CD Album" has all standard features like sequential or random playback, continuous (repeated) play, and a unique feature of rating your tracks. Ratings are stored in Windows registry, so each time you can choose songs of some average rating according to your mood and feelings.

0. System requirements

CD Album 1.5 uses Windows system codecs (decoders/encoders) to decode MP3 music. A standard MP3 decoder comes with Windows Media Player from Internet Explorer 4.0/5.0 (or higher). So if your system is Windows 98 or Internet Explorer is updated, then everything is OK. If you don't have Internet Explorer 4.0/5.0 (or higher), then please install it. Anyway, if you don't want to use MSIE, you can still run mpfull.exe from [setup] subdirectory (if you received our software on a CD) to install Windows Media Player.

If you create a CD album with WAV files coded in MPEG Layer 3 format, then you must distribute wav_mp3.exe in [setup] subdirectory, so that the program can install a necessary Windows system decoder.

1. How to play CD albums

If a CD album was correctly created to use with our software, then when the program starts, you will see a tree of albums' and songs' titles on the left. On the right there may be HTML pages displaying lyrics, pictures and additional information. If there is no HTML file for such information, a short description of the MPEG-3 file will appear.

Playback can be controlled from menu, toolbar and keyboard. As for the menu items and toolbar buttons, everything should be intuitive enough for you to use. We will give a short description for each button of the toolbar. From left to right buttons are:

Open: opens a new CD album in some directory of your disks or CDs

Locate: locates the current track's position in tree. When you are listening to a track, you can click the mouse on tree items and view other albums' and tracks' information. And, if you "get lost", then in order to return to the current track's position (and lyrics), you must use this command

Rating: sets a rating for selected track or album. If you clicked this button on an album's title, the rating you choose will be set for all tracks in the album.

Previous, Play, Pause, Stop, Next: standard buttons for playback

Repeat on/off: this turns repeat mode on/off (the recycled arrows mean repeat is ON)

Range of playback: one track, one album, all albums or shuffled (random) play

If you prefer to control playback from keyboard, you can use the following keys (we have implemented them a` la famous MPEG-3 player WinAMP):

Z/X/C/V/B are five keys which correspond to Prev/Play/Pause/Stop/Next on the button toolbar. Note that pressing [Enter] or double-clicking on a track's title will play that track just like pressing X. [Spacebar] can work for both Play and Pause buttons. If the player's not playing, it will start the playback. If the player's playing, it will toggle pause/resume.

D/A/T/S control the range of playback: whole [D]isc, one [A]lbum, one [T]rack or [S]huffled (random) play.

R controls [R]epeating. If playback's range is whole disc, one album or one track then that range will be repeated forever, until you change this mode. Only note that repeated play cannot be combined with shuffled random play. If random play is selected, then repeat option will be ignored.

L is for [L]ocating current track's position in tree.

O is for [O]pening a new album to play. It may be another disc or just a directory where its sub-directories contain MPEG-3 files. Please read further about creating CD albums to understand the structure your directory should have. For short, the album's directory is a set of sub-directories containing MPEG-3 files. It may have album.ini information file and HTML-designed pages, or not.

Gray +/-: When you are viewing songs' lyrics, the font size may be too small or too large for you. You can use "gray" keys on the right of your keyboard, plus (+) and minus (-) to change the font size. Please also see the context menu of the browser window by right-clicking any place in it. As you can see, text in browser window can also be selected and copied to clipboard. The Refresh function of the browser can be invoked by F5. This feature is especially useful when you are creating and designing your CD album (you can preview your work), a matter discussed in the next part of documentation .

"/" (slash) works for Ratings button. Note that when you specify some criteria for rated play, Prev/Next buttons will play previous or next tracks that suit the rating level. And repeated or shuffled play will also work only among the songs of a same or higher level of rating that you specified in Ratings window.

Esc: When you use ratings, sometimes the program cycles a long period of time to find next song that fits your rating criteria, so you might want to stop the process. If so, you can press ESC to stop it.

Special note: Ratings are turned off each time the program starts. Even though songs' ratings are stored in Windows registry, when you start the program, it will not use ratings, unless you open Ratings window and check "Use ratings during playback". This is not a bug, but a feature :o) because, being human, our tastes may vary from time to time. Sometimes we might want to listen to all songs, and sometimes only to very best ones. You can change the rating of some particular song any time you like, and this new rating will be automatically stored in Windows registry.

2. How to create a CD album

First, prepare your MPEG-3 files, using some "CD rippers" like CDex or MusicMatch Jukebox. Files playable in "CD Album" can have extension MP3 or WAV, but WAV files should be coded in MPEG Layer 3 format. Then, save files for a CD album in some directory, for example [Main], each musical album in its own subdirectory: [Main\Subdir1], [Main\Subdir2], etc.

Now in each sub-album's directory you must write a configuration file album.ini, from which "CD Album" will know the track titles and the title of the sub-album. You can use utility playlist.exe from our software pack. Just run the following command: 

playlist "directory-name"
or place playlist.exe into PATH and run playlist.exe from the sub-album's directory. In directory [Main] of the CD album, please place album.ini with information of the following kind: 
[general]
title=Christian Music Collection, Volume 1 (1999)
charset=204
Parameter charset specifies the character codepage for the program window. This is necessary in case you translate the program into another language by putting a new *.LNG (please see english.lng and russian.lng for an example). By default, charset=204, so if you don't need to translate the program to some language other than English or Russian, you can omit this parameter.

Track titles, specified in name parameter in sub-albums' album.ini, will appear in the albums/songs tree on the left. Physical path from path is the filename of MPEG-3 file with that track. In the sub-album's [general] section, you can also use a special feature of volume normalizing. Because MPEG-3 files you select for your CD album may come from many different sources, their volumes can differ greatly. But in album.ini you can normalize the volume by setting "volume=..." parameter for each album. By default, volume is 100 (percents). If the album is twice as loud as others, you can set volume=50. But if the album is 20% more quiet, you can set volume=125 (100/80=1.25).

HTML files for each album must be located in subdirectory [html] of the album's subdirectory, for example, [Main\Subdir1\html]. The main file must be named index.htm (you can have several files pointing to each other). If index.htm contains lyrics, you can force "CD Album" player to jump to current track's lyrics when it starts to play. For this purpose, lyrics of the track must be in index.htm with a link of type <a name="">. For example: 

<a name="3">
<h2>Lord, You are precious</h2>
Lord, You are more precious than silver<br>
Lord, You are more precious than gold
(3 if this is track number 3 in album).

That's all for creating files to use in "CD Album". To make a CD, you must have a CD Recorder. :o) In the root directory, please put the following files from our software pack: album.exe, album.htm, albumrus.htm, album.ico, autorun.inf, playlist.exe, and also album.ini, which is the configuration file you created for your CD album. Please also copy the subdirectory [setup], which contains setup files for Windows codecs that we talked about in section "0. System requirements."

And now... May be, the most interesting feature of "CD Album" is that, if root directory of the CD contains a subdirectory called [postcard] with a HTML page, it will be displayed at the top of this help file. So you can CUSTOMIZE each CD you create. For example, if you are preparing Christmas gifts for your friends, you can create for each of them a nice postcard in [postcard] with their names. So they'll be nicely surprised and no one will have the same copy of your Christmas album!
